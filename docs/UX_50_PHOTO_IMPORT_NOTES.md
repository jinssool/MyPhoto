# 50장 사진 가져오기 UX 안정화 메모

## 목적

실제 Google Drive 테스트 폴더에서 여러 장의 사진을 등록했을 때, 가족 사용자가 홈과 시간별 화면을 편하게 둘러볼 수 있는지 확인하기 위한 메모입니다. 이 작업은 새 기능 추가가 아니라 50장 규모에서 깨지는 화면을 안정화하는 데 초점을 둡니다.

## 확인된 문제

- Google Drive 썸네일을 바로 불러오지 못하는 경우 카드와 상세 화면에 깨진 이미지 표시가 보였습니다.
- 긴 파일 이름이 카드 제목, 상세 제목, 이전/다음 사진 버튼에서 줄바꿈 없이 밀리는 경우가 있었습니다.
- 가져온 사진 날짜가 원본 시간 문자열처럼 보여 가족 사용자가 읽기 어려웠습니다.
- 한 달에 사진이 많이 몰리면 시간별 화면의 작은 카드가 너무 촘촘해지고 일부 사진이 보이지 않는 문제가 있었습니다.
- 가져오기 화면에 `JSON`, `메타데이터` 같은 개발자용 표현이 남아 있었습니다.

## 이번에 고친 점

- 사진 카드와 상세 화면에 실패해도 깨지지 않는 이미지 대체 표시를 추가했습니다.
- 카드 이미지 비율을 고정하고 제목과 설명을 적당한 줄 수로 제한했습니다.
- 긴 파일 이름은 카드, 상세 정보, 이전/다음 링크에서 줄바꿈되도록 정리했습니다.
- 날짜 표시는 `2026년 5월 19일`처럼 가족이 읽기 쉬운 형식으로 바꿨습니다.
- 홈은 최근 사진 일부만 보여주고, 전체 사진은 시간별 화면으로 이어지도록 안내를 추가했습니다.
- 시간별 화면은 같은 달의 나머지 사진을 숨기지 않고 정돈된 격자로 보여주도록 바꿨습니다.
- 가져오기 화면은 미리보기와 앨범 등록의 차이가 더 잘 보이도록 문구를 다듬었습니다.

## 미룬 일

- 큰 폴더용 페이지네이션이나 무한 스크롤은 아직 넣지 않았습니다.
- 썸네일 프록시나 이미지 캐시는 만들지 않았습니다. 원본 사진을 앱에 저장하지 않는 MVP 원칙을 유지합니다.
- 사람 자동 인식, 지도, 댓글, 권한 기능은 이번 범위가 아닙니다.
- Google Drive 원본 파일은 계속 삭제, 이동, 수정하지 않습니다.

## 가족 테스트 때 볼 것

- 홈에서 사진이 너무 많아 보이지 않고 시간별 화면으로 자연스럽게 이동하는지 확인합니다.
- 시간별 화면에서 같은 달에 사진이 많이 있어도 원하는 사진을 찾을 수 있는지 봅니다.
- 깨진 썸네일 대신 파일 이름 대체 표시가 나와도 혼란스럽지 않은지 물어봅니다.
- 사진 상세 화면에서 좋아요, 숨기기, 둘러보기 제외, 다시 보이기 버튼의 의미가 분명한지 확인합니다.
- 가져오기 화면에서 “미리보기”와 “앨범에 등록하기”의 차이를 설명 없이 이해하는지 봅니다.
